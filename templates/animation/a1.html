Building a Custom Keyframing System in Unreal Engine 5

Assignment write-up by Kevin Land

In this assignment, our class was tasked to build our own keyframing system given an input text file where a key frame was denoted by a row of time, position, and axis-angle rotation. Here's how I did that in Unreal Engine 5:

1. Data Import

The best way I found to import the supplied text file was as a Data Table. Because UE5 requires CSV, I manually converted the .txt to a CSV (only 10 rows). I then built a Blueprint function to read rows from the Data Table.

2. Mapping t → u

I used a Timer to loop through each row and let the Tick function interpolate each frame. The timer delay is a variable Duration; u increments every Tick within that delay.

3. Interpolation

The naïve approach linearly interpolates both position and rotation:

To avoid gimbal lock, we instead need quaternion-based spherical linear interpolation (SLERP). I created two C++ Actor Components:

  • Lerp: homemade position interpolation.
  • Slerp: converts axis-angle (FRotator) → FQuat, normalizes, performs FQuat::Slerp, normalizes again, and converts back to FRotator.

4. Comparison

Finally I wired everything into the Blueprint so I could compare the built-in interpolation with my homemade method.
